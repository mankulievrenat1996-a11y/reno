# Дизайн: премиум-редизайн главной страницы («Elevated»)

Дата: 2026-07-18
Статус: утверждено пользователем, готово к планированию реализации

## Контекст

Идея пришла из стороннего текста про мульти-агентную AI-связку для «дорогого»
дизайна сайтов (тёмное стекло, неон, кинематографичные AI-медиа — в духе
higgsfield.ai/refero.design). Сайт «АкваЛайн» — B2B/B2C инженерная услуга
(ремонт водопроводов без отключения воды под давлением), уже прошедшая
несколько раундов премиум-полировки (см. `[[aqualine-site]]` в памяти):
дисплейный шрифт Space Grotesk, тёмные `.section--navy` секции со свечением
(`.hero::after`), магнитные кнопки, бегущая лента, каскадный reveal.

Задача — не копировать чужую тёмную tech-эстетику один в один (это сломало бы
доверительный инженерный тон бренда), а взять применимые техники (глубина,
многослойные тени, честные крупные фото, композиционная смелость) и усилить
уже существующую систему.

**Обсуждение шло через AI-компаньона в браузере** — без Node.js на машине
(стандартный browser-companion скилла не работает), поэтому макеты собирались
как самодостаточные HTML-файлы поверх реального `css/styles.css`, отданные
через уже запущенный локальный сервер `.claude/serve.ps1` на порту 5602.
Файлы удалены после решения (не часть репозитория).

**Ключевые решения по ходу брейншторма:**
- Масштаб тёмной темы: **светлый основной + тёмные акценты** (не весь сайт).
- Порядок работы: **сначала главная как пилот**, остальные 8 страниц — отдельным
  шагом после отдельного одобрения (uslugi.html, o-kompanii.html, proekty.html,
  kontakty.html, stati.html, 404.html, remont-bez-otklyucheniya-vody.html,
  stoimost-vrezki-pod-davleniem.html).
- Медиа: **реальные фото (Unsplash)**, без AI-генерации — для B2B-инженерной
  услуги честное фото объекта важнее «вау»-картинки.
- После 3 раундов сравнения макетов hero (изолированные фрагменты не давали
  пользователю уверенно оценить — see «не знаю»/«не нравится»/«no preference»)
  решение приняли на **полноценной копии страницы** (с шапкой/навигацией/
  плашкой доверия) — этот урок стоит повторить, если потребуется showcase
  других блоков.
- Между двумя полными вариантами (bento-сетка услуг vs. только elevation)
  выбран **Elevated без bento** — карточки услуг остаются равномерной сеткой,
  весь эффект — через глубину/тени/свечение.

## Дизайн-токены (расширение `:root` в `css/styles.css`)

Существующие токены (`--navy-*`, `--blue-*`, `--shadow-sm/md/lg`, `--r-sm/md/lg`,
`--font-display`) не меняются. Добавляются:

```css
--shadow-xs: 0 1px 2px rgba(10, 31, 68, .04);
--shadow-xl: 0 24px 48px rgba(10, 31, 68, .14);
--glass-bg: rgba(255,255,255,.72);
--glass-border: rgba(255,255,255,.5);
--glass-blur: blur(16px);
--navy-glow-blue: rgba(91,155,255,.20);
--navy-glow-teal: rgba(20,200,180,.14);
```

Используются точечно — не как замена существующей палитры.

## Hero: честное фото-карточка + стеклянная плашка-цифра

Композиция меняется с «фото на весь фон + стеклянная панель со списком»
на «фото — отдельная приподнятая карточка + один яркий факт крупным планом»:

- `.hero__grid` получает вариант с колонками `1.1fr .9fr` (было `1.05fr .95fr`) —
  структура (`hero__content` слева: бейдж/h1/подзаголовок/CTA/3 мелких статы)
  **не меняется**.
- Справа вместо `.hero__panel` (списка из 4 пунктов) — `.hero__media`:
  - `.hero__media-frame` — `assets/hero-pipeline.jpg` в рамке `aspect-ratio:4/5`,
    скругление `--r-lg`, тень `0 40px 80px rgba(0,0,0,.45)` + `inset` подсветка
    верхней грани.
  - `.hero__media-stat` — стеклянная плашка (`--glass-bg`/`--glass-blur`, но
    на тёмном фоне hero используется тёмное стекло `rgba(8,24,50,.68)`, не
    светлое — иначе не читается на фото), наплывает на нижний левый угол фото.
    Крупная градиентная цифра (`0`, «аварийных отключений воды за 12 лет
    работы») — единственный элемент такого масштаба на странице, задаёт
    фокус.
- Фоновая тёмная подложка hero (градиент + `.hero::before` сетка +
  `.hero::after` анимированное свечение) **не меняется** — уже соответствует
  направлению «тёмные акценты», выносить hero в светлую тему не нужно.
- Список из 4 пунктов («Производство не останавливает…», «Жители СНТ…»,
  «Не нужно сливать…», «Экономия времени…») по смыслу пересекается с уже
  существующим списком из 3 пунктов в блоке «Работа под давлением: вода
  продолжает течь» (`.split__list`, ~строка 1561 `index.html`: «Ноль
  простоя», «Меньше затрат», «Быстрее сроки»). Просто склеить в 7 пунктов
  нельзя — будет дублирование смысла («не сливать систему» / «меньше затрат»,
  «экономия времени» / «быстрее сроки»). Вместо этого `.split__list`
  дорабатывается до **4 пунктов**: существующие «Ноль простоя» и «Быстрее
  сроки» остаются как есть; «Меньше затрат» объединяется с «Не нужно сливать
  и заново запускать систему» в одну карточку; добавляется новый 4-й пункт
  на основе «Жители СНТ и посёлков не остаются без воды» (единственная мысль
  из hero-списка, не покрытая существующими тремя — акцент на человеческую,
  а не только промышленную аудиторию). Итог: 4 пункта без потери уникальных
  идей и без повторов.

## Depth-система «Elevated» (применяется по всей главной)

Без изменения структуры карточек/секций — только визуальная глубина:

**Карточки** (`.card`, `.service`, `.stat-card`, `.panel-card`):
- Базовая лёгкая тень в состоянии покоя (`--shadow-xs` уровня), на hover —
  составная тень (базовая + увеличенная + тонкое цветное кольцо
  `0 0 0 1px rgba(46,118,245,.10)`), подъём `translateY(-8px)` (было `-4/-6px`).

**Тёмные секции** (`.section--navy` как секция-обёртка встречается в
`index.html` дважды: «Как мы работаем» — 5 шагов, и «Технологический
партнёр» — карточки патентов; плюс сам `.hero` и footer уже тёмные, но их
фон/свечение не трогаем — правки только для двух `.section--navy`):
- Два декоративных радиальных свечения (`::before`/`::after`, синее и
  бирюзовое, `--navy-glow-blue`/`--navy-glow-teal`), низкая интенсивность,
  `position:absolute`, `overflow:hidden` на секции, `z-index` контент поверх.
- `.stat-card` на тёмном фоне получает `inset` подсветку верхней грани +
  усиленную рамку на hover (`rgba(91,155,255,.4)`).
- `.step__num` (кружки с цифрами шагов) получают дополнительное свечение-кольцо
  вокруг существующей тени.

**Кнопки:** `.btn--primary:hover` — существующая тень дополняется мягким
цветным ореолом (`0 0 40px rgba(46,118,245,.25)`).

**Явно не делаем на этом пилоте:** bento-сетка (асимметричные по размеру
карточки услуг/сфер применения) — пользователь отклонил после сравнения двух
полных вариантов; секции услуг и сфер применения остаются равномерной сеткой
как сейчас.

## Область изменений

**Файлы:** `css/styles.css` (источник правды для стилей) → пересборка через
`.claude/build.ps1` во все 9 `.html` (только `index.html` получит новую
разметку hero; у остальных 8 — только унаследованные CSS-правила, без
структурных изменений разметки на этом этапе, т.к. пилот ограничен главной).

**Не входит в этот этап:**
- Перенос дизайна на страницы, отличные от `index.html`.
- Любая AI-генерация изображений/видео.
- Полная тёмная тема.
- Bento-сетки.
- Замена/добавление реальных фото сверх уже используемого `hero-pipeline.jpg`.

## Тестирование

Ручная проверка в браузере (dev-сервер `.claude/serve.ps1`, порт из
`.claude/launch.json`):
- Hero: фото-карточка + стеклянная плашка читаемы и не наезжают на текст на
  desktop (1400px) и mobile (тест `resize_window` ≤480px — плашка уходит в
  статичное позиционирование под фото, см. медиа-запрос `max-width:900px`).
- Список 4 пунктов из старого hero не потерян — присутствует в блоке «Работа
  под давлением» без явного дублирования смысла с уже имеющимся там списком.
- Тени/свечения на карточках и тёмных секциях не «текут» за пределы контейнера
  (`overflow:hidden` на `.section--navy`), не портят читаемость текста.
- `prefers-reduced-motion` — существующее правило продолжает работать
  (свечения статичны по умолчанию, анимация есть только на `.hero::after`,
  уже учтено).
- Скриншот главной целиком (desktop + mobile) как подтверждение.
