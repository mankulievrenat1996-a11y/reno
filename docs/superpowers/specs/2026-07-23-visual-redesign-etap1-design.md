# Дизайн: визуальный редизайн, этап 1 (тёплая палитра — доводка)

Дата: 2026-07-23
Статус: утверждено пользователем, готово к реализации

## Контекст

Сайт «РПД» (см. `README.md`) только что перешёл с сине-белой темы на тёплую
терракотово-янтарную (вдохновение — tonisco.com, разбор конкурента r-power).
Смена цветов в `css/styles.css` и во всех 10 страницах уже сделана и
подтверждена рабочей (нет старых синих `rgb()`/hex ни в CSS, ни в инлайновых
SVG-иконках).

Пользователь запросил более широкий редизайн («дизайнерскую новую версию»).
Решение по декомпозиции (через superpowers:brainstorming): работа разбита на
два независимых этапа:

1. **Этап 1 (этот спек)** — визуальная доводка под новую палитру: не требует
   от пользователя никаких материалов, работаем с тем что уже на сайте.
2. **Этап 2 (отложено)** — контентное усиление доверия (реальные тех.детали
   в карточках проектов, реальные кейсы/клиенты вместо плейсхолдеров) —
   заблокировано до появления реальных данных от пользователя.

Варианты героя обсуждались через `mcp__visualize__show_widget` (интерактивные
HTML-макеты с кнопками `sendPrompt`) — визуальный компаньон
brainstorming-скилла недоступен (на машине не установлен Node.js, тот же
случай что и в `2026-07-17-form-feedback-and-404-design.md`).

## Дизайн

### 1. Герой — вариант 3 (правая панель «оживлена»)

Двухколоночная структура героя (`.hero__content` + `.hero__panel`) на
`index.html` сохраняется. Заголовок укрупняется (см. п.3). Правая панель
`.hero__panel` меняется: вместо статичного списка «Зачем нужна работа без
остановки потока» — визуальный блок с тёмным фоном (тёплый градиент
`--navy-800`→`--navy-900`), диагональным штрихом-паттерном (лёгкая текстура,
`repeating-linear-gradient`, ассоциация с металлом/трубой) и тремя плавающими
карточками-цифрами поверх (12+ лет, 800+ объектов, 24/7 выезд) — последняя
карточка (24/7) выделена акцентным (янтарным) фоном как самый «горячий»
факт. Текст-заголовок правой панели («Зачем нужна работа без остановки») и
подпись под ним остаются как есть — меняется именно нижняя часть панели:
список из 4 пунктов с галочками заменяется на визуальный блок с цифрами.
Сам список из панели убирается без переноса — секция «Преимущества» чуть
ниже на этой же странице (6 карточек: «Без остановки воды», «Аварийная
служба 24/7» и т.д.) уже покрывает тот же смысл, дублирование не нужно.

### 2. CTA-кнопки — янтарный акцент

`.btn--primary` (и любые основные кнопки-действия «Оставить заявку»,
«Заказать звонок», «Отправить заявку» и т.п.) — фон меняется с
терракотового градиента (`--blue-500`→`--blue-600`) на янтарный
(`--terra-500`→`--terra-600`). Текст, ссылки, бейджи, цифры и второстепенные
акценты остаются терракотовыми (`--blue-*` в текущих переменных). Это даёт
кнопке визуальный приоритет — единственный «горячий» жёлтый элемент на
странице, остальное — приглушённая медь.

`.btn--outline`, `.btn--light` — без изменений (нейтральные, не акцентные).

### 3. Типографика — укрупнение заголовков по всем страницам

Шрифтовая пара не меняется (`Inter` + `Space Grotesk` в `--font`/
`--font-display`). Меняется масштаб:
- `h1` (герой и `.page-hero` на внутренних страницах) — увеличить текущий
  `font-size` примерно на 15–20% и слегка сжать `line-height` для более
  плотного, «весомого» заголовка.
- `h2` (заголовки секций) — на внутренних страницах сейчас заметно мельче
  героя; довести до более уверенного контраста с body-текстом (сверка
  значений в `css/styles.css`, конкретные `px` — на этапе кода, не меняя
  переменные шрифта).

Изменение затрагивает только `font-size`/`line-height` правил в
`css/styles.css`, без новых шрифтов и файлов.

### 4. Ритм фона секций — чередование по всем 10 страницам

Ревизия порядка `<section>` на каждой странице: не должно идти подряд
несколько секций с одинаковым (белым) фоном без разбивки тёмным
(`.section--navy`/`.evidence`) или акцентным (`.section--sky`) блоком между
ними. Где нужно — переключить существующий модификатор класса секции
(`.section--sky`/`.section--navy`/без модификатора = белый), **не меняя
контент и порядок блоков**, только фон. Это применяется на всех 10 страницах
одинаково.

### 5. Карточки и иконки — полировка под тему

Точечная проверка (не переработка): обводки карточек (`.card`,
`.cards--terra .card:hover` и т.п.), фон плашек-иконок — привести к новой
палитре без «синеватых» полутонов, которые могли остаться от старых
`opacity`/`rgba`-миксов, не пойманных при замене палитры (например, если
где-то `border-color` смешивался с белым и давал холодный оттенок). Проверка
визуальная, по ходу работы над остальными пунктами.

## Технический подход

- Для всего, что меняется только в CSS (п.2–5) — источник правды
  `css/styles.css` (и `js/main.js`, если понадобится), сами `.html`
  напрямую не редактируются. После правок — `.claude/build.ps1` разносит
  изменения по всем 10 `.html` (проверено рабочим в этой сессии).
- П.1 (герой) требует также правки разметки `<body>` — она затрагивает
  только `index.html` (герой есть лишь на главной; внутренние страницы
  используют `.page-hero` без второй колонки), правится напрямую в файле,
  затем всё равно прогоняется через `.claude/build.ps1`, чтобы синхронизировать
  сопутствующие правки CSS.
- Проверка — через `preview_start({name: "site"})` (существующий
  `.claude/launch.json` → `.claude/serve.ps1`, порт 5602), не через ad-hoc
  `python -m http.server`.

## Вне рамок

- Реальные фото объектов, реальные клиенты/кейсы — Этап 2.
- Смена шрифтовой пары.
- Полный full-bleed фото-герой (вариант, который пользователь не выбрал).
