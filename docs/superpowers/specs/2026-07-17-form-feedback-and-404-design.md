# Дизайн: обратная связь формы заявки + страница 404

Дата: 2026-07-17
Статус: утверждено пользователем, готово к планированию реализации

## Контекст

Сайт «АкваЛайн» (статичный HTML/CSS/JS, см. `README.md`) имеет два незавершённых
"нулевых" (заглушечных) состояния:

1. **Обратная связь формы заявки** (`.form-success` в `css/styles.css`, логика в
   `js/main.js`) — сейчас успех и ошибка отправки показывают один и тот же
   зелёный блок с разным текстом. Ошибка выглядит как успех.
2. **Страница 404** — не существует. При переходе по несуществующему адресу
   после публикации сайта откроется стандартная страница хостинга без
   брендинга и без пути назад на сайт.

Формы (`form[data-form]`) встречаются на всех 9 HTML-страницах — как модалка
(дублируется в каждой странице) и как инлайн-форма (например, на `index.html`,
`kontakty.html`). Отправка идёт через FormSubmit (`fetch` в `main.js`), логика
`then`/`catch` централизована — правки JS-поведения делаются один раз в
`js/main.js`, затем разносятся по страницам через `.claude/build.ps1`
(инлайнит `css/styles.css` и `js/main.js` в каждый `.html`).

Варианты дизайна обсуждались через макет `mockups-zero-blocks.html`
(самодостаточный HTML-файл с интерактивными превью — использован вместо
браузерного компаньона brainstorming-скилла, т.к. на машине не установлен
Node.js). Пользователь выбрал:

- Форма → **вариант B** («Карточка вместо формы»)
- 404 → **вариант A** («Тёмный технический», в стиле hero-секции) + добавлен
  блок с телефоном

## Дизайн: обратная связь формы (вариант B)

**Успех:**
- Все поля формы и кнопка отправки скрываются.
- На их месте — центрированная карточка: зелёная круглая иконка-галочка,
  заголовок «Спасибо, заявка принята», подпись «Свяжемся с вами в ближайшее
  время», кнопка-CTA «На главную».

**Ошибка (сеть недоступна / FormSubmit не ответил):**
- Та же механика подмены, красная круглая иконка-предупреждение.
- Заголовок «Не получилось отправить», подпись.
- Кнопка «Позвонить» (`tel:+79818345864`, реальный номер сайта) — основное
  действие.
- Ссылка «Попробовать снова →» — возвращает исходную форму с уже введёнными
  значениями полей (ничего не сбрасывается), фокус на первое поле.

**Область действия:** все формы на сайте (модалка + инлайн), т.к. логика
централизована в `js/main.js` и применяется к любому `form[data-form]`.
Существующий bug-баннер honeypot-ловушки (тихий "успех" для ботов) продолжает
работать так же — использует ту же карточку успеха.

**Технический подход** (детали — на этапе плана реализации):
- Новая CSS-разметка карточки в `css/styles.css` (переиспользовать
  `--blue-*`, `--navy-*`, `--r-*`, `--shadow-*`, `--font-display` — как в
  остальном сайте; иконки — inline SVG `viewBox="0 0 24 24" stroke="currentColor"`
  в стиле уже используемых на сайте, не emoji и не иконный шрифт).
- Показ/скрытие исходных полей — CSS-класс-переключатель на `<form>`
  (например `form--result`), а не построение новой формы в JS: полей внутри
  формы касаться не нужно, они остаются в DOM и просто скрываются
  через `.form.form--result > *:not(.form-result) { display:none }`.
  Так сохраняются любые скрытые поля (honeypot, `data-service`-бейдж
  услуги в модалке — см. `js/main.js` про `modalServiceTag`) без доп. правок
  разметки на всех 9 страницах.
- Существующий `<div class="form-success">ТЕКСТ</div>` в каждой форме
  (2 инстанса на страницу × 9 страниц) заменяется на пустой контейнер
  `<div class="form-result"></div>`, содержимое (иконка/заголовок/текст/кнопки)
  генерируется в `js/main.js` при успехе/ошибке — единая точка правки текста
  вместо разночтений по 18 копиям.
- «Попробовать снова» — JS снимает класс `form--result`, восстанавливает
  кнопку отправки (текст/`disabled`), не трогает значения полей.
- Учесть `prefers-reduced-motion` при анимации появления карточки (сайт уже
  так делает в других местах, см. `main.js` про `reduceMotion`).

## Дизайн: страница 404 (вариант A + телефон)

Новый файл `404.html` в корне сайта, со стандартными шапкой/подвалом/модалкой
(как остальные 9 страниц — паттерн дублирования уже принят в проекте), но
без остальных секций. Основной контент:

- Тёмный градиентный фон (`--navy-800` → `--navy-900`, как `.hero`).
- Крупная цифра «404», градиент `--blue-400` → `--blue-500` (`background-clip: text`).
- Заголовок «Страница не найдена», короткая подпись.
- Две кнопки: «На главную» → `index.html`, «Наши услуги» → `uslugi.html`
  (подтверждено пользователем).
- Разделитель, ниже — блок «телефон в одно касание»: иконка трубки (inline
  SVG, стиль как у существующих иконок сайта) + кликабельный номер
  `+7 (981) 834-58-64` (`tel:+79818345864` — на десктопе клик копирует номер,
  на мобильном звонит, поведение уже реализовано глобально в `main.js` для
  всех `a[href^="tel:"]`) + подпись «Не нашли нужное? Просто позвоните».

**Адаптив:** на узких экранах цифра и заголовок уменьшаются, кнопки
переходят в столбик, телефонный блок не расползается по ширине (проверено
переключателем ширины в `mockups-zero-blocks.html`).

**Ограничение:** статичный `404.html` открывается по прямому URL, но чтобы
хостинг реально показывал именно эту страницу при 404-ошибке (а не свою
стандартную), после выбора хостинга понадобится настройка на его стороне
(`ErrorDocument 404 /404.html` для Apache, `error_page 404 /404.html` для
Nginx, либо аналог в панели хостинга). Публикация ещё не сделана (см.
`ПЛАН-ПУБЛИКАЦИИ.md`) — эта настройка войдёт в шаги публикации, не в текущую
задачу.

## Вне рамок текущей задачи

- Состояние «ничего не найдено» для фильтров/поиска — на сайте нет ни
  одного фильтра или поиска (`proekty.html` — статичная сетка), делать не
  для чего.
- Настройка хостинга на показ `404.html` при реальной 404-ошибке —
  относится к публикации сайта, не к дизайну/вёрстке.
- Другие "нулевые" состояния (не упомянутые пользователем) в рамки не
  входят.

## Затронутые файлы (ожидается на этапе реализации)

- `css/styles.css` — новые классы карточки результата формы, стили 404.
- `js/main.js` — переписать submit-хендлер (успех/ошибка → карточка,
  retry-ссылка), общая логика для телефонных ссылок не меняется (уже
  работает).
- `404.html` — новый файл (шапка/футер/модалка скопированы по образцу
  существующих страниц).
- Все 9 существующих `.html` — `<div class="form-success">` → `<div class="form-result"></div>`
  (18 инстансов), затем `.claude/build.ps1` пересобирает инлайн CSS/JS.
- `sitemap.xml` — 404.html **не** добавляется (страница ошибки не должна
  индексироваться); стоит проверить, что `robots.txt` не блокирует её
  случайно (не обязана быть в disallow, но и не должна быть в sitemap).
